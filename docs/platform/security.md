# 安全与透明

## 给普通用户

- 使用 **Escrow**：对方先锁定资金，你再开工  
- **不要** 把助记词给任何人，包括「客服」  
- 核对 DApp 域名与合约地址（官方 GitHub 公布）  
- 主网前建议在测试网练习  

## 给 Skill / Agent 创造者

- Skill 描述如实，避免误导性承诺  
- 高价 Skill 建议从小额交易积累信誉（v0.2）  
- 敏感模型可链下加密交付，链上只存 hash  

## 给企业

- 企业钱包建议 **多签**  
- API 密钥放在企业网关，不写入链上  
- 签署数据处理协议（DPA）  
- 可要求第三方渗透测试 + 合约审计报告  

## 给开发者

- 遵循 [开源与安全](/developers/open-source-security)  
- 依赖及时更新，CI 跑测试  
- 负责任披露：security@（主网前公布邮箱）  

## 平台承诺

| 承诺 | 状态 |
|------|------|
| 合约开源 | 私有开发中，产品可用后公开 |
| 主网前审计 | 计划 v1.0 前 |
| Bug Bounty | 计划随主网 |
| 费率链上可见 | MVP 已做到 |
| 非托管 | 架构级保证 |
