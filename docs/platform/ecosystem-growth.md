# 生态壮大：控制权、门槛与激励

VibeAgent 采用 **渐进式去中心化**：早期保留升级与费率控制权以快速迭代；通过 **低门槛 SDK** 与 **交易费模型** 吸引开发者、设备方与资本，再逐步移交 DAO。

## 三类核心目标

1. **保留平台控制权** — 智能合约可升级（UUPS）、任务治理、早期 Sequencer  
2. **降低入场门槛** — 开源 Agent/IoT 模板，有 Python 就能接入  
3. **可扩展治理** — 锁仓释放、MasterChef 激励、未来链上投票  

## 谁如何参与？

| 对象 | 激励 | 你怎么做 |
|------|------|----------|
| **投资者** | 协议增长、代币增值（若发行） | 早期轮 + 锁仓；关注 [投资者叙事](/vision/investors) |
| **开发者 / Agent 创作者** | 交易分成、流量 | 用 SDK 发布 Skill/Agent/数据服务 |
| **合作公司** | 品牌、联合方案 | 设备白名单、行业 Oracle、联合推广 |
| **普通用户** | 低费率、返利 | wallet/worker 发单接单；推荐奖励 |
| **IoT 厂商** | 设备上链收入 | 认证设备接入数据/充电/能源市场 |

## 平台怎么赚钱？

| 收入 | 场景 |
|------|------|
| Escrow / 市场协议费 | 任务、Skill、人类众包 |
| L2 Gas（早期） | 全链交易，尤其微额高频 |
| 数据调用费 | 传感器 × 每秒百万次 |
| 能源清算费 | 每度电链上结算抽成 |
| 企业契约 Gas | 冷链等大单条件执行 |

战略转向：**用交易费养生态**，而非仅靠代币补贴流动性。
